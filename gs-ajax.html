AJAX

.ajax()

Ova metoda šalje asinhroni ili sinhroni HTTP(Ajax) zahtev.

Različiti načini da se pozove funkcija ajax()

- Prosleđivanjem URL-a nad kojim se vrši zahtev i određene opcije

.ajax(URL, {...opcije...})

- Samo prosleđivanjem određenih opcija

.ajax( {...opcije...})

Neke od potrebnih i često korišćenih opcija su:

  • url - određuje URL kome se šalje zahtev
  • async - (podrazumevano: true) određuje da li se vrši asinhroni ili sinhroni zahtev
  • data - podaci koji se šalju serveru (u slučaju POST zahteva)
  • method - (podrazumevano: 'GET') metod HTTP zahteva (može biti 'GET', 'POST', 'PUT')
  • username - username koji se koristi ako server zahteva autentifikaciju
  • password - šifra koja se koristi ako server zahteva autentifikaciju
  • statusCode - (podrazumevano: {}) objekat u kome se na osnovu numeričkog status koda koji server vrati poziva odgovarajuća funkcija
  • dataType - tip podatka koji očekujete od servera (npr. xml, html, script, json, text)
U datom primeru smo kreirali dugme na čiji klik se šalje GET zahtev koji traži od servera određeni .json fajl (kreiran radi testiranja ovog primera). Zahtev vraća podatke u JSON formatu i potrebno ih je prebaciti u JavaScript objekat pomoću funkcije JSON.stringify(). Podatke smo na kraju dodali na stranicu u vidu paragrafa.

.get()

Ova metoda šalje asinhroni ili sinhroni HTTP GET zahtev. GET zahtev se šalje kada hoćete da preuzmete neke podatke sa servera.

Različiti načini da se pozove funkcija get()

- Prosleđivanjem URL-a nad kojim se vrši zahtev, podataka koji se šalju sa zahtevom, callback funkcije koja se pokreće kad se zahtev uspešno izvrši (sa sobom donosi i podatke od servera i status kod zahteva) i tip podataka koji očekujete

.get(URL, podaci, callback funkcija, tip podataka)

- Samo prosleđivanjem određenih opcija

.get( {...opcije...})

.get() je uglavnom ekvivalent jednoj ovakvoj .ajax() funkciji

$.ajax({
    url: url,
    data: data,
    success: success,
    dataType: dataType
});

Neke od potrebnih i često korišćenih opcija su:

  • url - određuje URL kome se šalje zahtev
  • async - (podrazumevano: true) određuje da li se vrši asinhroni ili sinhroni zahtev
  • data - podaci koji se šalju serveru (u slučaju POST zahteva)
  • method - (podrazumevano: 'GET') metod HTTP zahteva (može biti 'GET', 'POST', 'PUT')
  • username - username koji se koristi ako server zahteva autentifikaciju
  • password - šifra koja se koristi ako server zahteva autentifikaciju
  • statusCode - (podrazumevano: {}) objekat u kome se na osnovu numeričkog status koda koji server vrati poziva odgovarajuća funkcija
  • dataType - tip podatka koji očekuejte od servera (npr. xml, html, script, json, text)
U datom primeru smo kreirali dugme na čiji klik se šalje GET zahtev koji traži od servera određeni .json fajl (kreiran radi testiranja ovog primera). Zahtev vraća podatke u JSON formatu i potrebno ih je prebaciti u JavaScript objekat pomoću funkcije JSON.stringify(). Podatke smo na kraju dodali na stranicu u vidu paragrafa.

.post()

Ova metoda šalje asinhroni ili sinhroni HTTP POST zahtev. POST zahtev se šalje kada hoćete da postavite neke podatke na server.

Različiti načini da se pozove funkcija post()

- Prosleđivanjem URL-a nad kojim se vrši zahtev podataka koji se šalju sa zahtevom, callback funkcije koja se pokreće kad se zahtev uspešno izvrši (sa sobom donosi i podatke od servera i status kod zahteva) i tip podataka koji očekujete

.post(URL, podaci, callback funkcija, tip podataka)

- Samo prosleđivanjem određenih opcija

.post( {...opcije...})

.post() je uglavnom ekvivalent jednoj ovakvoj .ajax() funkciji

$.ajax({     type: "POST",
    url: url,
    data: data,
    success: success,
    dataType: dataType
});

Neke od potrebnih i često korišćenih opcija su:

  • url - određuje URL kome se šalje zahtev
  • async - (podrazumevano: true) određuje da li se vrši asinhroni ili sinhroni zahtev
  • data - podaci koji se šalju serveru (u slučaju POST zahteva)
  • method - (podrazumevano: 'GET') metod HTTP zahteva (može biti 'GET', 'POST', 'PUT')
  • username - username koji se koristi ako server zahteva autentifikaciju
  • password - šifra koja se koristi ako server zahteva autentifikaciju
  • statusCode - (podrazumevano: {}) objekat u kome se na osnovu numeričkog status koda koji server vrati poziva odgovarajuća funkcija
  • dataType - tip podatka koji očekuejte od servera (npr. xml, html, script, json, text)
U datom primeru smo kreirali tri input polja za tekst i jedno dugme. Klikom na dugme šaljemo POST zahtev određenoj stranici, u našem slučaju post_example.php koji smo kreirali u svrhe testiranja, i sa zahtevom šaljemo i podatke iz input elemenata. Stranica dobija podatke iz input polja i izvrši neke izmene nad njima i šalje podatke nazad. Ovaj određeni primer uzima i me, prezime godinu rođenja i vraća poruku u formatu "Zovem se [ime] [prezime] i imam (2021 - [godiste]) godina/e/u.".
